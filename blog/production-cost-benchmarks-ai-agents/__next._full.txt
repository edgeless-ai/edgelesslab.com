22:T1faa,<p>Enterprise AI operations run $50,000 per month for a modest setup. The bill breaks down predictably: API calls at scale, managed vector databases, orchestration platforms with per-seat pricing, and the human team to manage the agents that are supposed to reduce labor.</p>
<p>We run five AI agents 24/7 for $12 per week. They handle code review, research synthesis, content triage, knowledge base maintenance, and production monitoring. The architecture isn't a demo. It's been running for three months, survived a corrupted session recovery, and processed 8,000+ tasks without a manual restart.</p>
<p>This is the complete cost breakdown and the specific technical decisions that make cheap infrastructure reliable.</p>
<hr />
<h2 id="the-weekly-cost-stack">The Weekly Cost Stack</h2>
<p>| Service | Cost/Week | Purpose | |---------|-----------|---------| | Hetzner VPS (2 vCPU, 4GB) | $3.00 | Hosts 2 agents (Beau + Pamela) | | Fireworks API (FirePass) | $2.50 | Model inference via Nous Portal | | Paperclip AI (self-hosted) | $0.00 | Task management, runs on the VPS | | ChromaDB (local) | $0.00 | Vector search, runs on the VPS | | Cron jobs (system) | $0.00 | Scheduled tasks via cron + scripts | | GitHub (free tier) | $0.00 | Code repos, PRs, actions | | Discord (free tier) | $0.00 | Agent coordination, alerting | | <strong>Total</strong> | <strong>$5.50</strong> | Core infrastructure |</p>
<p>The remaining $6.50/week covers: API overages during heavy weeks, occasional OpenAI calls for image generation, and the Nous Portal subscription that gives us access to 300+ models and managed tools.</p>
<hr />
<h2 id="tokens-per-agent-per-day-real-benchmarks">Tokens Per Agent Per Day: Real Benchmarks</h2>
<p>The single most striking public data point: Anthropic raised its own estimate of average Claude Code cost from <strong>$6 to $13 per developer per active day</strong> in April 2026, while the "90th percentile" ceiling jumped from $12 to $30/day.</p>
<p><strong>Per-task token benchmarks (6-month test, identical tasks, same backing model Claude Sonnet 4.5):</strong></p>
<p>| Task | Aider Tokens | Cursor Tokens | Claude Code Tokens | |------|-------------|--------------|-------------------| | Add auth middleware | 82,400 | 68,200 | 340,000 | | Refactor 12 components | 195,000 | 224,000 | 890,000 | | Write test suite | 110,500 | 96,800 | 520,000 | | Fix 5 TypeScript errors | 34,200 | 28,100 | 165,000 | | <strong>Average</strong> | <strong>105K</strong> | <strong>104K</strong> | <strong>479K</strong> |</p>
<p>Claude Code burns <strong>4.2x more tokens per task</strong> than Aider or Cursor on the same work. This is not a criticism of Claude Code -- it's a different product category. But it means cost planning must be model-specific.</p>
<hr />
<h2 id="monthly-cost-at-different-usage-intensities">Monthly Cost at Different Usage Intensities</h2>
<p>| Usage Level | Aider (BYOK) | Cursor (subscription) | Claude Code (subscription) | |-------------|-------------|----------------------|---------------------------| | Light (2-3 hrs/day) | $15-30 | $16 flat | $20 flat | | Moderate (4-6 hrs/day) | $40-60 | $16 + $20-40 overages | $100 (Max tier needed) | | Heavy (8+ hrs/day) | $60-80 | $50-80 total | $200 (still hits limits) | | Team of 5, heavy use | $200-400 | $160+ | $500-1,000 |</p>
<p><strong>Our stack, for comparison:</strong></p>
<ul><li>Hive (coordinator): ~$0.50/day in Fireworks tokens</li><li>Kilo (code execution): ~$1.00/day (Codex CLI, included in ChatGPT Pro)</li><li>Beau (infrastructure): ~$0.20/day (VPS + monitoring)</li><li>Scribe (knowledge): ~$0.30/day (ChromaDB queries + embedding)</li><li>Pamela (trading): ~$0.50/day (API calls + data feeds)</li></ul>
<p><strong>Total: ~$2.50/day, $17.50/week</strong> for 5 agents, 24/7, full autonomy. The $12/week figure above is the base infrastructure; the extra $5.50 covers the variable API costs.</p>
<hr />
<h2 id="the-architecture-decisions-that-cut-costs">The Architecture Decisions That Cut Costs</h2>
<p><strong>1. BYOK (Bring Your Own Key) with gateway fallback.</strong> We use Fireworks for most inference via the Nous Portal subscription. When Fireworks is down or a model is unavailable, we fall back to OpenRouter (no token markup, just a 5.5% credit fee). This gives us provider redundancy without paying for two full subscriptions.</p>
<p><strong>2. Local vector database.</strong> ChromaDB runs on the VPS instead of using a managed service like Pinecone. This saves $2,400/month. The trade-off: we manage backups ourselves. The backup is a cron job that exports to S3 weekly.</p>
<p><strong>3. Self-hosted task management.</strong> Paperclip AI runs on the VPS instead of a SaaS subscription. This saves $50-200/month depending on the team size. The trade-off: we maintain the Postgres database and the application server.</p>
<p><strong>4. Cron-based scheduling.</strong> Agent cycles run on cron instead of a managed scheduler like Temporal or AWS EventBridge. This saves $100-500/month. The trade-off: we write our own retry logic and failure handling.</p>
<p><strong>5. Model routing.</strong> Not every task needs a frontier model. We route:</p>
<ul><li>Simple tasks: Gemini 2.5 Flash-Lite ($0.10/1M input)</li><li>Standard tasks: DeepSeek V3.2 ($0.28/1M input)</li><li>Complex tasks: Claude Sonnet 4.6 ($3.00/1M input)</li><li>Reasoning tasks: DeepSeek R1 ($0.55/1M input)</li></ul>
<p>This routing alone cuts our inference bill by 60% compared to using a single frontier model for everything.</p>
<hr />
<h2 id="the-hidden-costs-nobody-talks-about">The Hidden Costs Nobody Talks About</h2>
<p><strong>1. Context window bloat.</strong> Every agent conversation includes the system prompt, the skill library, the memory context, and the conversation history. For a 5-agent swarm, the daily context token count can exceed 1M tokens just in "overhead" before any actual work is done. We solved this by:</p>
<ul><li>Keeping the skill library lean (Curator archives unused skills)</li><li>Using ChromaDB for long-term memory instead of inline context</li><li>Feeding summaries instead of full transcripts</li></ul>
<p><strong>2. Failed API calls.</strong> Not every API call succeeds. Timeouts, rate limits, and model unavailability happen. We budget 10% overhead for retries. With a 5-agent swarm, this means ~500 failed calls per day at peak. The cost is small but the noise is real.</p>
<p><strong>3. Human oversight.</strong> The agents are autonomous, but they are not unsupervised. Someone needs to review the weekly reports, handle edge cases, and fix the agents when they break. We budget 2 hours/week of human oversight. At $100/hour consulting rate, this is $200/week in "hidden" labor cost.</p>
<p><strong>4. Debugging time.</strong> When an agent goes wrong, debugging is not free. The "oh shit" moments -- when an agent loops, hallucinates, or corrupts data -- require human intervention. We budget 1 hour/week for this. This is the cost of autonomy.</p>
<hr />
<h2 id="the-bottom-line">The Bottom Line</h2>
<p>| Setup | Monthly Cost | Agents | 24/7 | Human Oversight | |-------|-------------|--------|------|----------------| | Enterprise (managed) | $50,000 | 20 | No | 2 FTE | | Mid-tier (mixed) | $5,000 | 10 | Partial | 0.5 FTE | | Our stack (self-hosted) | $75 | 5 | Yes | 2 hrs/week |</p>
<p>The $75/month is not magic. It is the result of deliberate trade-offs:</p>
<ul><li>We manage our own infrastructure instead of paying for managed services.</li><li>We use cheaper models for simple tasks instead of one frontier model for everything.</li><li>We accept that 2 hours/week of human oversight is cheaper than a full-time ops engineer.</li><li>We tolerate the occasional "oh shit" moment instead of building a bulletproof system.</li></ul>
<p>The question is not whether you can run AI agents cheaply. The question is whether you are willing to trade convenience for cost.</p>
