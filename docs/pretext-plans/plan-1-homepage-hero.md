# PreText Phase 1 -- homepage obstacle-aware hero prototype

Goal:
Build one signature PreText moment on the homepage that feels unmistakably Edgeless.

Target:
- homepage hero only

Concept:
Route the main headline and supporting text around one compositional object instead of a normal two-column block.

Candidate objects:
- generative geometry
- product constellation map
- abstract operating-system fragment
- orbital or sculptural form

Actions:
1. Define 2-3 hero copy variants.
2. Prototype three layouts:
   - current/default
   - CSS-balanced
   - PreText obstacle-aware
3. Test desktop, tablet, and mobile behavior.
4. Compare legibility, distinctiveness, and visual rhythm.
5. Choose the strongest variant or reject the experiment.

Deliverables:
- prototype comparison note
- screenshots or preview links
- exact recommendation: keep, revise, or reject

Acceptance criteria:
- the PreText variant feels materially more Edgeless than the CSS-only version
- it remains legible and stable across breakpoints
- it does not create a puzzle or gimmick feeling

Decision gate:
If the PreText hero is only marginally better than the CSS version, stop here and keep the simpler implementation.
