# PreText Phase 2 -- editorial Lab/About prototype

Goal:
Use PreText in one longform/editorial context where it can create a genuinely authored reading experience.

Target:
- one featured Lab page or the About page

Concept:
Let text wrap around diagrams, chips, timeline fragments, or system objects.

Possible insertions:
- architecture chips
- role labels
- studio capability map
- inline tool stack ribbons
- diagram fragments or screenshots

Actions:
1. Pick one page only.
2. Define the narrative object the text should wrap around.
3. Prototype one restrained layout and one more ambitious layout.
4. Test reading flow, not just aesthetics.
5. Identify whether the treatment improves comprehension and tone.

Deliverables:
- one editorial prototype
- reading-flow critique
- recommendation: apply to selected longform pages only / reject / revise

Acceptance criteria:
- page feels more authored and high-design
- reading quality improves, not just novelty
- layout stays stable on mobile

Decision gate:
If the layout feels performative rather than useful, reject it.
