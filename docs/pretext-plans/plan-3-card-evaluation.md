# PreText Phase 3 -- product and lab card evaluation

Goal:
Decide whether product and lab cards gain enough from PreText to justify the complexity.

Targets:
- Products page cards
- selected Lab cards

Concept:
Use PreText only if it materially improves multiline title wrapping, teaser density, and scanability.

Actions:
1. Select a representative set of cards with uneven title lengths.
2. Build side-by-side comparisons:
   - tuned CSS version
   - PreText version
3. Evaluate density, clarity, and scanning speed.
4. Note whether metadata and teaser flow improve.
5. Decide if this should enter the design system.

Deliverables:
- comparison note
- recommendation per card type
- explicit list of where PreText is worth it and where it is overkill

Acceptance criteria:
- measurable or visibly clear improvement in scanability
- no maintenance burden that outweighs the gain
- recommendation is unambiguous

Decision gate:
If the gain is subtle, keep CSS and do not expand PreText into cards.
