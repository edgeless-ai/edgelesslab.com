# Pre-Launch Checklist

Complete every item before publishing your Gumroad product.

## Content

- [ ] Main guide is written and proofread
- [ ] All code/templates tested on a clean machine
- [ ] README.md is under 500 words and gets buyer to value in < 5 min
- [ ] CHANGELOG.md exists with v1.0.0 entry
- [ ] Examples directory included (at least 1 worked example)
- [ ] File and directory names are clean (no spaces, no special characters)
- [ ] ZIP unpacks into a single named directory (no loose root files)

## Listing

- [ ] Short description written (under 150 characters, action-oriented)
- [ ] Full description follows pain-credential-contents formula
- [ ] "Who this is NOT for" section included
- [ ] Price justification anchors against buyer's time
- [ ] Cover image uploaded (1280x1280 PNG, minimal design)
- [ ] Price set and matches your pricing worksheet decision

## Cross-Sell & Links

- [ ] 2-3 related products linked in the listing
- [ ] 2-3 related products linked in the README inside the ZIP
- [ ] All Gumroad URLs verified with `scripts/verify-links.sh` (HTTP 200)
- [ ] No dead links in any file inside the ZIP

## Distribution

- [ ] Blog post written or drafted (links to product page)
- [ ] Social post or thread drafted (provides value, mentions product)
- [ ] Product page URL added to your website/portfolio
- [ ] Existing products updated to cross-link to the new one

## Final Checks

- [ ] Downloaded the product yourself and tested the full buyer experience
- [ ] Opened every file in the ZIP to confirm nothing is broken
- [ ] Searched the ZIP for placeholder text ("[TODO]", "XXX", "REPLACE")
- [ ] Asked one person to review the listing copy before publish
- [ ] Launch day is Tuesday or Wednesday (not Monday or Friday)

---

**After publish:** Do not change pricing for 30 days. Monitor conversions for 7 days before editing the listing. Respond to buyer messages within 24 hours.
