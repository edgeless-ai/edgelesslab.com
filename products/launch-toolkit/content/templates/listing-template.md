# Gumroad Listing Template

Use this template to write your product listing. Fill in each section, then paste into Gumroad's product editor.

---

## Short Description (150 characters max)

_This appears in search results and embeds. State what the product DOES, not what it IS._

```
[Action verb] [specific pain]. [Number] [deliverable type] for [audience], ready to [use case].
```

**Example:** Stop fighting ESLint configs. 14 battle-tested rule sets for TypeScript, ready to drop in.

## Full Description

### Pain (2-3 sentences)

_Name the specific problem. Be concrete. Use numbers and scenarios, not vague frustration._

```
You have spent [time] trying to [painful task]. Every [attempt] leads to [specific bad outcome].
[Relatable scenario that your buyer has lived through].
```

**Example:** You have spent a full weekend configuring Kubernetes resource limits only to watch your staging cluster OOM on Monday morning. The defaults are wrong for production. The docs assume you already know which values matter.

**Your pain:**
> ___


### Credential (1-2 sentences)

_Why should they trust you? Show scope and duration, not title._

```
Built from [duration] of [activity] across [scope].
```

**Example:** Built from 2 years of running Kubernetes in production across 12 services handling 50k+ requests/minute.

**Your credential:**
> ___


### Contents (bulleted list)

_Exactly what is in the download. File names, counts, specifics._

```
- [filename] -- [what it does] ([size/count detail])
- [filename] -- [what it does] ([size/count detail])
- [filename] -- [what it does] ([size/count detail])
```

**Example:**
- `resource-limits.yaml` -- Production-ready resource configs for 8 common workload types (API, worker, cron, etc.)
- `guide.md` -- 4,200-word guide explaining every value choice with reasoning
- `monitoring/` -- 3 Prometheus alert rules that catch resource misconfigs before they cause outages
- `README.md` -- 2-minute setup walkthrough
- `CHANGELOG.md` -- Version history

**Your contents:**
> ___


### Who This Is NOT For

_Disqualify bad-fit buyers. This builds trust and reduces refunds._

```
This is not for you if:
- [Condition that means they do not need this]
- [Condition that means a different product fits better]
- [Condition that signals mismatched expectations]
```

**Example:**
This is not for you if:
- You are running a single-node dev cluster and do not plan to go to production
- You are using a managed Kubernetes platform that handles resource allocation for you
- You want a full Kubernetes course (this is a config toolkit, not a tutorial)

**Your "not for":**
> ___


### Price Justification (1 line)

_Anchor against the buyer's time, not competing products._

```
Less than [time comparison to buyer's rate], saves you [concrete time saved].
```

**Example:** Less than 30 minutes of consulting time. Saves you a full weekend of trial-and-error configuration.

**Your price justification:**
> ___

## Cross-Sell Section

_Add to the bottom of your listing. Link 2-3 related products._

```
### Related Products

- **[Product Name]** ($XX) -- [One-sentence description of what it does and why the buyer would want it next]
- **[Product Name]** ($XX) -- [One-sentence description]
```

---

## Pre-Publish Checklist

- [ ] Short description is under 150 characters
- [ ] Pain section names a specific, concrete problem
- [ ] Credential shows scope/duration, not just title
- [ ] Contents list matches the actual files in the ZIP
- [ ] "Not for" section included
- [ ] Price justification anchors against buyer's time
- [ ] Cross-sell links point to live Gumroad URLs
- [ ] Cover image uploaded (1280x1280, not AI-generated looking)
