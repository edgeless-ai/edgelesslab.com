# Dot-Matrix Cylinder Scroll

## Concept

A CSS 3D scroll-driven cylinder where 12 "text items" are living 16×16 dot-grid canvases running Tixy mathematical formulas. No letters, no words — only generative code art arranged in a tunnel that rotates with scroll.

One of the 12 grids is a **ghost grid**: a live camera feed rendered as a 16×16 brightness dot-matrix portrait in lime green. The user's face becomes a biological intrusion into a purely computational space.

## Technique Notes

### 1. Tixy Dot-Grid Rendering
Each of the 12 cylinder items is a `<canvas>` element (256×256px, 16×16 dots at 16px each). Each canvas runs a different preset formula `(t, i, x, y) => value` evaluated every frame. Positive values render white dots; negative values render red dots; magnitude controls radius. The formulas are stored as strings and evaluated via `new Function("t", "i", "x", "y", "with (Math) { return " + code + " }")`.

### 2. CSS 3D Cylinder Positioning
A wrapper `<section>` gets `perspective: 70vw`. The `<ul>` gets `transform-style: preserve-3d`. Each `<li>` is `position: absolute` with `backface-visibility: hidden`. The 12 items are positioned using `sin/cos * radius` math: `radius = min(window.innerWidth, window.innerHeight) * 0.4`, `spacing = 180 / 12`. Each item gets `translate3d(0, sin(angle)*radius, cos(angle)*radius) rotateX(angle)`.

### 3. Scroll-Driven Rotation
The `<ul>` wrapper rotates on the X-axis from `0deg` to `180deg` as the user scrolls through a 300vh pinned section. The section uses `position: sticky` with a tall spacer (`height: 300vh`). Scroll position is normalized and mapped to rotation via `targetRotation = scrollProgress * 180`.

### 4. Scroll Momentum
The actual rotation is not locked to scroll position. It uses a lerp: `currentRotation += (targetRotation - currentRotation) * 0.05`. When the user stops scrolling, the cylinder continues to coast for 1–2 seconds, as if it has mass.

### 5. Independent Canvas Animation
Each of the 12 canvases updates via its own `requestAnimationFrame` loop at 60fps, independent of scroll. The `t` value is `(Date.now() - startTime) / 1000`.

### 6. Formula Inheritance
Clicking any canvas copies its formula from the grid currently facing the camera (the front grid). The clicked grid "learns" from the leader. If the front grid is the ghost camera, the clicked grid advances to its next preset instead. This creates a social behavior across the generative art.

### 7. Ghost Grid
Grid #7 (index 6) is not a Tixy formula grid. It is a live camera feed (`getUserMedia`) rendered as a 16×16 brightness grid. The webcam is sampled every frame, each pixel is converted to grayscale, and the dot radius is mapped to brightness. If the camera is denied, it falls back to a static lime sine wave.

## How to Run

1. Open `index.html` in any modern web browser.
2. No build step, no external assets, no server required.
3. The camera feature is optional — if denied, the ghost grid falls back to a static lime sine wave.
4. Scroll to rotate the cylinder. Move the mouse for parallax. Click any grid to copy the front grid's formula.
