# Elastic Shader Type — BEND

A self-contained, interactive typographic demo that renders the word **BEND** entirely inside a WebGL fragment shader using 2D signed-distance fields (SDFs). GSAP elastic physics drives shader uniforms with tactile overshoot and bounce. The demo is a single HTML file that loads Three.js and GSAP from CDNs — no build step, no external assets.

## What It Does

- **SDF Typography**: The letters B, E, N, and D are built from primitive SDF operations (boxes, circles, line segments) and arranged horizontally. A `smoothstep` anti-aliases the edges.
- **3D Extrusion Fake**: The word is rendered twice — once as a dark offset shadow (driven by `u_extrusion`) and once as the main white fill — creating a convincing 2.5D look without any 3D geometry.
- **GSAP Elastic Physics**: When a letter is activated, GSAP tweens `u_spacing` (letter gap), `u_extrusion` (shadow depth), and `u_twist` (sine-wave distortion) with `elastic.out` and `back.out` eases. The overshoot creates a bouncy, tactile personality.
- **Scroll-Driven Color Blocks**: The page has four 100vh sections (coral, teal, lemon, deep teal). `IntersectionObserver` detects the active section and activates the corresponding letter.
- **Mouse Parallax**: Normalized mouse coordinates are fed to `u_mouse`, adding subtle tilt to the extrusion shadow.
- **Click Interaction**: Clicking anywhere on the canvas triggers the elastic tween for the letter in that horizontal zone.
- **Escaped Dot (3% Weirdness)**: A small gold SDF circle orbits the word in a Lissajous curve (frequency ratio 1.3:0.7). It is autonomous — it does not respond to GSAP, mouse, or scroll — and leaves a faint trail. It represents the "escaped punctuation" of a letter that has broken free and become a celestial body.

## Technique Notes

- **No 3D geometry**: The entire scene is a single `PlaneGeometry(2, 2)` with an `OrthographicCamera` and a custom `ShaderMaterial`. All rendering happens in the fragment shader.
- **No font files**: The letters are hardcoded SDF functions inside the shader.
- **Transparent canvas**: The WebGL renderer uses `alpha: true` so the colored DOM sections show through the background.
- **Per-letter opacity**: The shader uses `u_activeLetter` (float 0–3) and approximate X-coordinate boundaries to dim inactive letters to 30% opacity.
- **Responsive**: The canvas and `u_resolution` uniform resize automatically.

## How to Run

1. Double-click `index.html` to open it in any modern browser.
2. Or serve it with a local static server:
   ```bash
   cd ~/claude-projects/creative-demos/elastic-shader-type
   python3 -m http.server 8000
   # Then open http://localhost:8000
   ```
3. Scroll through the four sections to trigger elastic transitions.
4. Click anywhere on the word to trigger a letter-specific elastic deformation.
