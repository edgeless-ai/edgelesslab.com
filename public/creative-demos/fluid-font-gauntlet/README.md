# Fluid Font Gauntlet

A real-time WebGL Navier-Stokes fluid simulation drives variable-font axes
(`wght`, `wdth`, `slnt`) in a 6×6 typographic specimen grid.

## How to Run

Open `index.html` in any modern browser. No build step or server required.

## Interactions

- **Mouse move / touch drag**: Injects velocity and dye into the fluid.
- **Click / tap**: Cycles dye color (cyan → magenta → yellow).
- **Scroll wheel**: Adjusts dye decay (`dissipation`).
- **Hover a cell**: Freezes its axis values and shows a tooltip.
- **Key 'R'**: Resets the fluid and re-randomizes ghost cells.

## Technique Notes

- WebGL 1.0 fullscreen ping-pong FBO pipeline with 8 hand-written GLSL shaders:
  advection, curl, vorticity confinement, divergence, pressure Jacobi,
  gradient subtract, splat, and display.
- Simulation runs at 256×256 resolution. Dye is read back via `gl.readPixels`
  once per frame and mapped to `font-variation-settings` for each cell.
- 3% of cells are "ghost" doppelgängers: they sample from the mirror-opposite
  cell and reveal a burgundy border when pointer velocity exceeds 0.15.
- Tactile echo: rapid axis changes (>150 combined delta) inject a 50ms cyan
  text-shadow imprint.
- Overlay architecture: WebGL canvas at `z-index: 0`, DOM grid at `z-index: 1`
  with transparent backgrounds.

## Simplifications

- `slnt` (slant) is used instead of `ital` because Roboto Flex does not expose
  an `ital` variable axis.
- Braille dividers (0.5% weirdness) are implemented as standard 1px `#222222`
  gaps instead of Unicode Braille dot patterns, which would require sub-pixel
  layout beyond the gap width.
