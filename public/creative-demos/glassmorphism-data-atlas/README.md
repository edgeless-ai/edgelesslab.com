# Glassmorphism Data Atlas

A standalone, interactive HTML demo fusing **Atlas Card glassmorphism** with a **Pudding-style D3.js scatter plot** driven by scroll.

## What it does

- **Fixed D3.js background**: A dark scatter plot of 150 synthetic hip-hop data points (unique words vs. albums) rendered in a fixed SVG layer behind the UI.
- **Scroll-scrubbed zoom camera**: As you scroll down the 900vh runway, the D3.js zoom interpolates between 6 narrative waypoints, panning and zooming cinematically to different genre clusters.
- **Glassmorphism panels**: Floating frosted panels (`backdrop-filter: blur(36px)`) with gold-to-white text-gradient headlines appear and fade as you approach their scroll thresholds.
- **Velocity blur**: Fast scrolling increases the panel blur up to `72px` and dims them slightly, creating a physical resistance effect.
- **Mirror points**: ~3% of the data points (5 out of 150) exist as invisible drifting orbs inside the glass panels. When they touch the panel edge, they "reflect" into the scatter plot as cyan hollow circles that expand, linger, and fade.
- **Gold veins**: Hovering over a panel reveals faint gold connecting lines between the data points that fall behind that panel, as if the glass is a selective lens.
- **Light theme leak**: After 85% scroll progress, a warm white band slowly floods up from the bottom of the screen, irreversibly brightening the void. The glass panels react by becoming slightly more opaque.

## Technique Notes

### Architecture
- **Overlay separation**: The D3.js SVG is `position: fixed; z-index: 0`. The glass panels are `position: sticky` inside tall spacer divs (`z-index: 1`). A canvas overlay sits at `z-index: 2` for gold veins. This is the production-standard GPU-compositing pattern used by both Lusion and Atlas Card.
- **Single-file delivery**: Everything—CSS, HTML, JS—is inline in `index.html`. The only external dependency is D3.js (v7.8.5) from a CDN, plus Google Fonts for Inter and Space Mono.
- **No build step**: Double-click `index.html` to run in any modern browser.

### Zoom Interpolation
D3.js zoom transforms are objects with `.x`, `.y`, and `.k` properties. The scroll position `t = scrollY / maxScroll` is mapped to the nearest pair of narrative waypoints. Linear interpolation between the bracketing transforms is applied every frame via `requestAnimationFrame`, giving buttery smooth camera motion without D3's built-in transition engine.

### Glassmorphism CSS
```css
backdrop-filter: blur(36px);
background: linear-gradient(180deg, rgba(255,255,255,0.15) 0%, rgba(255,255,255,0.02) 100%);
border: 1px solid rgba(255,255,255,0.12);
box-shadow: 0 8px 32px rgba(0,0,0,0.4);
```
The headline uses `background-clip: text` with a `#D4AF37` → `#FFFFFF` gradient for a metallic gold effect.

### Performance
- The heavy work per frame is O(n) mirror-point drift and O(n) vein drawing. With n=150 this is trivial.
- D3 zoom is updated directly each frame with `svg.call(zoom.transform, ...)`, bypassing D3's interpolation engine.
- `will-change: transform, opacity, backdrop-filter` is set on panels to promote them to their own compositor layers.

### Simplifications
- The gold vein drawing uses screen-projected coordinates (`d3.zoomTransform` applied to scales) rather than a full quadtree spatial index. For 150 points this is more than fast enough.
- The mirror point orbs are small DOM divs inside the panels rather than canvas-drawn sprites. This keeps the implementation simple and readable.
- The axis grid lines are static; they do not re-render during zoom. In a full production build, axes would be re-ticked and redrawn to match the visible data domain.
- The dataset is fully synthetic (randomized names and values). No real artist data is used.
