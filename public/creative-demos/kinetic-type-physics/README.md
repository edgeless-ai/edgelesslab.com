# Kinetic Type Physics

A self-contained, interactive physics playground where every keystroke spawns a
letter that falls, bounces, and collides in a **Matter.js** world. Each letter
carries a lightweight inline **Lottie** animation that plays on impact. The demo
fuses two creative-code traditions:

- **Animography** — modular, per-character animated typefaces
- **Resn** — playful DOM-physics mini-experiences using Matter.js

---

## How to Run

Open `index.html` in any modern web browser. No build step, no server, and no
external assets other than the two CDN scripts (Matter.js and Lottie Web).

Because the Lottie animations are generated inline as JSON objects, the demo
works fully offline after the first page load.

---

## Interaction Design

| Interaction | Behavior |
|-------------|----------|
| **Type A–Z** | Spawns a 60×60px letter at the top-center with random horizontal velocity. The letter falls under gravity, bounces, and collides. |
| **Drag** | Matter.js `MouseConstraint` lets you grab and throw any letter. |
| **Click empty space** | Clears all letters and resets the physics world. |
| **Collision** | When two letters collide, both play their Lottie animation. A brief green flash (`#00FF7F`) appears at the impact point. |
| **Idle** | Letters settle on the floor. |

## Technique Notes

### 1. Inline Lottie Generator

Rather than relying on external Lottie JSON URLs (which may rot or require
CORS), each letter animation is synthesized on the fly by `generateLottieData()`.
The function builds a valid Lottie 5.5.2 JSON object with a single shape layer
whose transform animates in one of four templates:

- **Bounce** — scale 0 → 130 % → 100 %
- **Spin + Bounce** — scale 0 → 140 % → 100 % with a full 360° rotation
- **Wobble** — asymmetric scale 0 → 150/80 → 80/140 → 100/100 with a rocking rotation
- **Pulse** — multi-step scale 0 → 120 → 90 → 110 → 100

The shape itself cycles through ellipse, square, and rounded rectangle based on
the letter’s alphabet position. Because the template and shape both depend on the
character index, swapping two letters’ animations produces a visible change in
motion signature.

### 2. Matter.js + DOM Sync

Each letter is a **Matter.js** `Bodies.rectangle` with `restitution: 0.6` for a
playful bounce. Every frame the engine steps forward, and the corresponding
DOM wrapper is moved via `transform: translate(...) rotate(...)` to match the
body’s position and angle. This is the classic "physics-to-DOM" bridge pattern
popularized by Resn and Dropbear.

### 3. Collision-Triggered Animation

The `collisionStart` event listener checks every colliding pair. If a pair
contains a letter body, that letter’s Lottie instance is kicked with
`goToAndPlay(0)`. A tiny 16 px green circle is spawned at the collision point
and removed after 100 ms.

### 4. Rogue Glyph (3 % Weirdness)

Every 600 frames (10 seconds at 60 fps) one existing letter is promoted to a
**Rogue Glyph**:

- **Gravity immunity** — `body.gravityScale` is set to `0` and its velocity is
  overridden every frame with a Lissajous orbit: `vx = sin(t*2)*3`,
  `vy = cos(t*1.5)*3`.
- **Visual mutation** — color switches to coral red (`#FF6B6B`) and a glowing
  `drop-shadow` filter is applied.
- **Animation swap** — when the rogue comes within 40 px of the cursor, it
  searches for the nearest non-rogue letter within 60 px. If it finds one, the
  two letters’ Lottie animation instances are swapped, both play immediately,
  and after 3 seconds the rogue is deactivated and the animations are reverted.

This creates a temporary "typographic virus" — a letter that is not what it
appears to be.

### 5. Color Palette

The letters cycle through three Resn-style neon colors based on alphabet
position modulo 3:

- White `#FFFFFF`
- Magenta `#FF00FF`
- Yellow `#FFFF00`
