# Liquid Decryption

A hybrid interactive demo fusing **terminal kinetic typography** (staggered glyph scramble) with **Cuberto-style fluid advection** (noise-driven velocity field + mouse stirring). Built as a single self-contained HTML file using p5.js.

---

## Technique Notes

### Fluid Grid
- **64×64 pixel grid** covering the canvas. Each cell stores `{vx, vy}`.
- Velocities are driven by **multi-octave Simplex/Perlin noise** (`noise()` with spatial offsets and time evolution) producing an organic, ever-shifting flow.
- **Mouse injection** applies a radial force to cells within a 5-cell radius (~80 cells) of the cursor, making the liquid stir responsively without looping all 4096 cells.
- **Scroll boost** amplifies the global noise magnitude; fast scrolling makes the field swirl violently. The boost decays exponentially each frame.
- A **subtle trail overlay** (`#111111` at low alpha) renders cells with speed > 0.5, giving depth to the fluid motion. Toggle with `S`.

### Glyph Scramble & Decay
- Each keystroke spawns a **glyph particle** near the center.
- The glyph starts as a random symbol from `!@#$%^&*()_+-=[]{}|;:,.<>?` in phosphor green (`#00FF41`).
- **Staggered exponential decay**: each new glyph adds 5 frames to the base delay, producing a cascading left-to-right decryption rhythm.
- Color interpolates smoothly toward white (`#FFFFFF`) as the timer drops.
- Once resolved, the glyph continues to be advected by the fluid field.

### Advection & Edge Wrapping
- Glyphs sample the grid cell beneath them and inherit half the cell velocity.
- **Friction** (0.95) keeps speeds bounded.
- **Edge wrapping** ensures particles never drift permanently off-screen; they reappear on the opposite edge.

### Ghost Vortex (3% Weirdness)
- Exactly 3% of keystrokes spawn an **invisible ghost glyph** instead of a normal letter.
- The ghost has no text, no color, and no scramble sequence.
- However, it plants a **persistent vortex** lasting 600 frames (10 seconds) at its spawn point.
- The vortex exerts a **tangential force** (`2.0 / distance`) on all glyphs within 200px, causing them to orbit the empty center like water draining into an invisible hole.
- The only visual evidence is:
  1. **Swirling motion** of surrounding glyphs around an empty point.
  2. A faint **coral pulse** (`#FF6B6B` at 5–12% opacity) at the vortex center, but **only** when the mouse cursor is within 50px of it.

## Interaction

| Action | Behavior |
|--------|----------|
| **Type** | Spawns a glyph particle near center. Starts scrambled, decays into the correct letter. |
| **Mouse move** | Injects radial force into the fluid grid; glyphs swirl around the cursor. |
| **Scroll** | Amplifies the global fluid amplitude (`scrollBoost`). Fast scrolling = violent swirl. |
| **R** | Clears all glyphs and vortices; resets fluid grid. |
| **S** | Toggles the fluid trail overlay (`#111111` rectangles). |

## Architecture

The code is organized into explicit modular functions matching the Animation System Specification:

1. `initFluidGrid()` — Fluid Grid Module
2. `updateFluid()` — Noise-driven velocity evolution
3. `injectMouseForce()` — Mouse stir (radial injection)
4. `drawTrails()` — Trail Renderer
5. `spawnGlyph()` — Glyph Spawn Module
6. `updateScramble()` — Scramble Decay Module
7. `advectGlyph()` — Advection Module
8. `applyVortices()` — Ghost Vortex Module

## Performance

- Glyph cap: **300** (FIFO eviction)
- Ghost cap: **10** (FIFO eviction)
- Mouse injection: optimized to ~80 cells instead of 4096
- No external assets; loads p5.js from CDN
- Runs at 60fps in modern browsers
